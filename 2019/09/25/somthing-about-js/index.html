0 干啥呢？

最近要面试一个前端的岗位，正好复（yu）习（xi）一下相关的知识。其实我一直管自己叫“半吊子前端”，因为一直没有系统的学习过知识，而是用什么学什么。

1 JavaScript

1.1 null vs. undefined vs. NaN

会被转换为false情况

  • undefined
  • null
  • false
  • 0
  • NaN
  • ""或''

不会被转换为false的情况

  • {}
  • []
  • …

1.2 == vs. ===

=== 会比较类型

1.3 void 运算符

是执行一个表达式，然后不返回任何值 / return undefined

1.4 包装对象

new一个Number、String、Boolean的话，会返回object

不new直接赋值，不会

1.5 this

  • 全局，this === window
  • 构造函数，对象
  • 对象的方法，有时候是该方法的上一级context，有时候是运行的context（e.g. window）

如何指定函数内部this？

1.5.1 Function.prototype.call()

1.5.2Function.prototype.apply()

1.5.3 Function.prototype.bind()

将函数体内的this绑定到某个对象，然后返回一个新函数

1.6 prototype

类似C++类里的static。

1.7 'use strict'

  1. 只读属性不可写

  2. 函数不能有重名的参数

  3. 禁止 this 关键字指向全局对象

  4. 禁止删除变量
  5. 禁止使用 with 语句
  6. 创设 eval 作用域

1.8 异步

1.8.1 事件循环 Event Loop

为了检查异步任务是否执行完毕，引擎在不停地检查，一遍又一遍。只要同步任务执行完了，引擎就会去检查那些挂起来的异步任务。

1.8.2 操作模式

  1. 回调 callback

    cons: high coupling

  2. 事件监听

  3. observer pattern

1.8.3 Promise

promise 的状态

  • 异步操作未完成（pending）
  • 异步操作成功（fulfilled）
  • 异步操作失败（rejected）

3 HTTP and HTTPS

3.1 POST vs. GET

  • GET把数据包含在URL中，POST通过request body传递数据

  • GET在浏览器回退时是无害的，POST会再次提交请求

  • GET请求在URL中传送有长度限制，POST没有

  • GET只接受ASCII字符，POST没有限制
