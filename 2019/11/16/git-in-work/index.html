scenario

重做 commit

背景

每次开发一个新的功能，需要从 master 分支切一个 feat 分支，在 feat 分支上开发完成后，再 merge 到 master。

假设

你切错了分支，在一个脏分支上做了一些 commit。你需要把这些 commit 重做到干净的分支上。应该怎么办？

解决方案

  1. 重做一个 commit
    git cherry-pick

  2. 重做一段 commit
    git rebase

背景

当前分支有不想 commit 的改动，但有需要切换到别的分支。

解决方案

git stash
